## C. 必改项（P0）

### C-1：10-文档远程仓库章节与实际操作不同步

**问题描述**：`docs/10-开发准备与Sprint0.md` Section 1.2 写的是：
- 建议地址格式 `git@github.com:<用户名>/seat-reservation.git`
- "当前状态：❌ 尚未推送（待补远端推送）"

但实际已执行的操作：
- 远端已配置为 `https://github.com/hyc-i/ava.git`（HTTPS，非 SSH）
- 推送尝试后因无 credential helper 认证超时失败

**修复方案**：将 Section 1.2 更新为实际远端地址和实际失败原因：

```
### 1.2 远程仓库

**远端地址**：`https://github.com/hyc-i/ava.git`（HTTPS）

**当前状态**：⚠️ 首次推送未完成。本地 commit `c5998ba` 已就绪，`git push` 因 GitHub HTTPS 认证超时失败
（当前 CLI 环境未配置 credential helper / Personal Access Token）。

**后续操作**：
- 方式一（推荐）：在本地终端配置 GitHub PAT 后执行 `git push -u origin main`
- 方式二：改用 SSH：`git remote set-url origin git@github.com:hyc-i/ava.git && git push -u origin main`
```

### C-2：远端推送未完成

**问题描述**：远端仓库 `https://github.com/hyc-i/ava.git` 为空。GitHub HTTPS 推送因无 credential helper 认证超时。

**修复方案**：

```bash
# 方式一：配置 Personal Access Token（推荐）
git remote set-url origin https://<TOKEN>@github.com/hyc-i/ava.git
git push -u origin main

# 方式二：在已登录 GitHub 的终端中执行
git push -u origin main
```

## D. 建议优化项（P1~P2）

### D-1：任务板与 10-文档 Sprint 0 任务编号对齐

**问题描述**：`10-文档` Section 4.2 的 Sprint 0 任务清单（T00-01~T00-11）与 `项目任务板.md` Sprint 0 分区（T00-01~T00-10）存在两处不一致：

| 编号 | 10-文档 4.2 | 任务板 Sprint 0 分区 | 差异 |
|------|-------------|---------------------|------|
| T00-01 | 创建解决方案 .sln | 创建解决方案 .sln **和** Web 项目 .csproj | 任务板合并了两项 |
| T00-02 | 创建 Web 项目 .csproj | 安装 NuGet 包 | 任务板偏移一项 |
| — | (无此任务) | T00-09 验证 dotnet build + dotnet run | 任务板新增一项 |
| T00-11 | [可选] 推送远端 | 合并到 T00-10 | 任务板合并了 Git 和推送 |

**建议**：统一编号，例如在 10-文档 4.2 中也合并 .sln + .csproj 为一项，并增加"验证 build+run"为正式任务项，使两文档 T00-01~T00-10 一一对应。

### D-2：10-文档 Section 4.3 完成标准 ↔ 实际完成状态

**问题描述**：Section 4.3 Sprint 0 完成标准中 5 条全部用 ✅ 标记为预期，但其中第 5 条"Git 本地仓库已初始化并提交"实际已完成，第 1~4 条（build/run/建表/种子）尚未完成。所有 ✅ 均为预填，未区分已完成和待完成。

**建议**：将第 5 条改为实际完成状态（如添加"✅ 本地仓库已初始化并提交（commit c5998ba）"），其余 4 条改为 ⬜ 以反映真实状态。

### D-3：README "当前阶段" 与 10-文档 "Sprint 0" 重复描述

**问题描述**：README 的"当前阶段"（设计文档→代码过渡期）和 10-文档的 "Sprint 0 定义" 内容高度重叠，但分散在两份文档中。单人开发时容易忘记同步更新两者。

**建议**：README 的"当前阶段"改为指向 10-文档的链接（如"详细 Sprint 规划见 `docs/10-开发准备与Sprint0.md`"），不再在 README 中重复 Sprint 0 任务细节。

### D-4：任务板 Sprint 数量配置说明

**问题描述**：任务板预设了 Sprint 0~6 共 7 个分区。如果实际开发中需要调整 Sprint 数量或分裂某个 Sprint，当前任务板模板没有预留弹性（如"添加新 Sprint 分区"的指引）。

**建议**：在任务板的"任务卡编号规则"下方或文档末尾增加一条说明：

```
> **弹性调整**：如果某个 Sprint 发现工作量偏大，可将其轮次扩展（如在 Sprint 2 中增加到 3 轮），或将其中一个轮次拆分为独立 Sprint（需更新编号规则并重新排序任务卡）。
```
