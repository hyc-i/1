## 1. 关键链路选择说明

### 1.1 链路来源

前序文档已明确两条主链路：

| 链路 | 来源 | 步骤数 | 涉及页面 | 涉及 Controller |
|------|------|--------|----------|-----------------|
| **链路一：用户预约主链路** | 04-4.2 用户主链路 | 7 步 | P-U01 ~ P-U05 共 5 页 | HomeController, SeatsController, ReservationsController |
| **链路二：管理处理主链路** | 04-5.2 管理主链路 | 4 步 | P-A01 ~ P-A04 共 4 页 | AdminController |

### 1.2 前序文档已明确的边界

| 边界 | 来源 |
|------|------|
| 学生身份由 Session 管理，不建表 | 08-2, 07-9.1 |
| 管理员身份由 Cookie Authentication 管理 | 07-9.2 |
| 座位状态实时计算，不存字段 | 08-6.2, 04-6.1 |
| 预约状态仅两个值：`预约中` → `已取消`（不可逆） | 08-6.1, 04-6.2 |
| 冲突检测由应用层完成，唯一索引无法防御时段重叠 | 08-7.3, 08-9.3 |
| 统计口径区分：首页"当前占用" vs 统计页"当日总量" | 08-8.2 |
| 外键 DeleteBehavior.Restrict，座位用软删除 | 08-9.4 |
| Controller 不写业务逻辑，Service 不抛业务异常 | 07-4.4, 07-8.2 |
| 错误返回使用 `(bool Success, string ErrorMessage)` 元组模式 | 07-8.2 |

## 2. 链路一：页面与路由

### 2.1 链路概览

```
  【起点】学生打开浏览器访问系统
      |
      v
  [P-U01] 用户首页（未登录）
      |  POST /Home/Login  体验登录
      v
  [P-U01] 用户首页（已登录）
      |  点击"选座预约"
      v
  [P-U02] 座位列表页
      |  点击空闲座位"查看"
      v
  [P-U03] 座位详情页
      |  点击"立即预约"
      v
  [P-U04] 预约提交页
      |  POST 提交
      |
      +--[成功]--> [P-U05] 我的预约页
      |
      +--[失败]--> [P-U04] 预约提交页（停留，显示错误）
      |
      v

      |  点击"取消"
      v

      |
      v

```

| 步骤 | 页面 | HTTP | 路由 | 前置条件 | 后置页面 |
|------|------|------|------|----------|----------|
| L1-1 | P-U01 用户首页 | POST | `/Home/Login` | 未登录 | P-U01 用户首页（已登录） |

| L1-4 | P-U03 座位详情页 | GET | `/Seats/Details/{id}` | 已登录，来自列表 | P-U04 预约提交页 |
| L1-5 | P-U04 预约提交页 | GET | `/Reservations/Create?seatId={id}` | 已登录，座位存在且当前可预约 | L1-5 POST |
| L1-5 | P-U04 预约提交页 | POST | `/Reservations/Create` | 已登录，ModelState 有效 | P-U05（成功）/ 同页（失败） |

| — | P-U01 用户首页 | POST | `/Home/Logout` | 已登录 | P-U01 用户首页（未登录） |

### 2.3 各步骤详细说明

#### L1-1：体验登录

| 属性 | 值 |
|------|-----|
| 触发方式 | 未登录首页展示 3 个体验按钮（学生 A / 学生 B / 学生 C），点击任一 |
| 请求 | `POST /Home/Login`，body: `name=学生A` |
| 页面表现 | 写入 Session，页面刷新为已登录状态 |
| 不需要校验 | 用户名固定 3 个，不存在非法输入 |
| 成功后跳转 | `RedirectToAction("Index", "Home")` → P-U01 已登录 |

#### L1-2：首页展示（已登录）

| 属性 | 值 |
|------|-----|
| 触发方式 | 登录后自动跳转，或点击导航"首页" |
| 请求 | `GET /Home/Index` |
| 页面内容 | 欢迎语"欢迎，学生 A"、3 个统计卡片（总座位数/空闲/已预约）、"选座预约"入口、"我的预约"入口 |

**空闲座位数 = 总座位数 − 当前被占用的座位数**（实时，与统计页口径不同）

#### L1-3：座位列表

| 属性 | 值 |
|------|-----|
| 触发方式 | 点击"选座预约" |
| 请求 | `GET /Seats/Index?area=三楼自习区`（area 可选，不传则显示全部） |
| 页面内容 | 区域下拉筛选（全部/一楼大厅/二楼阅览区/三楼自习区）、座位卡片列表（编号/区域/描述/状态） |
| 座位状态 | 实时计算：当前时间有预约中记录 → 橙色"已预约"，否则绿色"空闲" |

#### L1-4：座位详情

| 属性 | 值 |
|------|-----|
| 触发方式 | 点击空闲座位的"查看"按钮 |
| 请求 | `GET /Seats/Details/{id}`，如 `/Seats/Details/5` |
| 前置校验 | 座位 ID 必须存在且 IsActive = true，否则返回 404 视图 |
| 页面内容 | 座位编号、区域、楼层、设施描述、实时状态（可预约/已预约）、"立即预约"按钮（仅空闲时显示） |

#### L1-5：预约提交

| 属性 | 值 |
|------|-----|
| 触发方式（GET） | 点击座位详情页的"立即预约" |
| 请求（GET） | `GET /Reservations/Create?seatId={id}` |
| GET 校验 | seatId 对应的座位必须存在、IsActive、当前空闲，否则 Redirect 到列表页 |
| GET 返回 | 表单页面：座位号（只读显示）、日期输入（默认明天）、开始时间选择、结束时间选择 |
| 触发方式（POST） | 填写完毕点击"提交" |
| 请求（POST） | `POST /Reservations/Create`，body: `seatId, reservationDate, startTime, endTime` |
| 后置校验 | ModelState 验证（必填、日期格式、EndTime > StartTime、未来时间） |
| 业务校验 | 冲突检测（同座位 + 同时段 + 预约中） |
| 成功 | 写入 DB → `RedirectToAction("My")` |
| 失败 | `ModelState.AddModelError` → 返回当前 View 显示错误 |

**时间验证规则：**
- `ReservationDate` 必须 >= 明天（不可预约当天或过去）
- `StartTime` 和 `EndTime` 在同一天（不跨天）
- `EndTime - StartTime` 必须 > 0
- 不限制最小预约时长（演示场景信任用户）

#### L1-6：我的预约

| 属性 | 值 |
|------|-----|
| 触发方式 | 导航点击"我的预约"，或预约成功自动跳转 |
| 请求 | `GET /Reservations/My` |
| 页面内容 | 该学生的所有预约记录列表，每条显示：座位号、区域、日期、时段、状态、"取消"按钮（仅"预约中"的记录可取消） |
| 排序 | 按 CreatedAt DESC（最新在上） |
| 空状态 | 无预约记录时显示空状态卡片 + "去选座"按钮 |

#### L1-7：取消预约

| 属性 | 值 |
|------|-----|
| 触发方式 | 点击"预约中"记录的"取消"按钮 → 确认弹窗 → 确认 |
| 请求 | `POST /Reservations/Cancel/{id}` |
| 前置校验 | 预约 ID 存在；Status = '预约中'（防止重复取消）；StudentIdentifier 匹配（归属校验） |
| 通过 | Status → '已取消', CancelledAt → 当前时间 UTC → `RedirectToAction("My")` |
| 不通过 | `TempData["ErrorMessage"]` + `RedirectToAction("My")`，刷新后在页面顶部显示错误提示 |

```csharp
[HttpPost]
[ValidateAntiForgeryToken]
public async Task<IActionResult> Create(ReservationCreateViewModel model)
{
    if (!_studentContext.IsLoggedIn)
        return RedirectToAction("Index", "Home");

    if (!ModelState.IsValid)
    {
        // 重新填充只读的 SeatNumber
        var seat = await _seatService.GetSeatDetailAsync(model.SeatId);
        model.SeatNumber = seat?.SeatNumber ?? "未知";
        return View(model);
    }

    var (success, errorMsg) = await _reservationService.CreateReservationAsync(
        model.SeatId,
        _studentContext.CurrentStudentName,
        _studentContext.CurrentStudentIdentifier,
        model.ReservationDate,
        model.StartTime,
        model.EndTime
    );

    if (!success)
    {
        ModelState.AddModelError("", errorMsg);
        var seat = await _seatService.GetSeatDetailAsync(model.SeatId);
        model.SeatNumber = seat?.SeatNumber ?? "未知";
        return View(model);
    }

    return RedirectToAction("My");
}
```

### 4.5 关键 Service 伪代码

**CreateReservationAsync 冲突检测逻辑：**

```csharp
public async Task<(bool Success, string ErrorMessage)> CreateReservationAsync(
    int seatId, string studentName, string studentIdentifier,
    DateTime date, DateTime startTime, DateTime endTime)
{
    // 1. 检查座位是否存在且活跃
    var seat = await _context.Seats.FindAsync(seatId);
    if (seat == null || !seat.IsActive)
        return (false, "该座位不存在或已停用");

    // 2. 冲突检测：同一座位 + 同一天 + 时段重叠 + 预约中
    var conflict = await _context.Reservations.AnyAsync(r =>
        r.SeatId == seatId &&
        r.ReservationDate == date.Date &&
        r.Status == "预约中" &&
        r.StartTime < endTime &&
        r.EndTime > startTime);

    if (conflict)
        return (false, "该时段已被预约，请选择其他时段");

    // 3. 创建预约记录
    var reservation = new Reservation
    {
        SeatId = seatId,
        StudentName = studentName,
        StudentIdentifier = studentIdentifier,
        ReservationDate = date.Date,
        StartTime = startTime,
        EndTime = endTime,
        Status = "预约中",
        CreatedAt = DateTime.UtcNow
    };

    _context.Reservations.Add(reservation);
    await _context.SaveChangesAsync();
    return (true, null);
}
```

**CancelMyReservationAsync 归属校验逻辑：**

```csharp
public async Task<(bool Success, string ErrorMessage)> CancelMyReservationAsync(
    int reservationId, string studentIdentifier)
{
    var reservation = await _context.Reservations.FindAsync(reservationId);

    if (reservation == null)
        return (false, "预约记录不存在");

    if (reservation.StudentIdentifier != studentIdentifier)
        return (false, "无权取消他人的预约");

    if (reservation.Status != "预约中")
        return (false, "该预约已取消，无需重复操作");

    reservation.Status = "已取消";
    reservation.CancelledAt = DateTime.UtcNow;
    await _context.SaveChangesAsync();
    return (true, null);
}
```

## 7. 链路一：状态流转

### 7.1 状态变化对应表

| 业务动作 | 作用对象 | 从状态 | 到状态 | 附加变化 | 触发 Service |
|----------|----------|--------|--------|----------|-------------|
| 学生提交预约成功 | 预约记录（Reservation） | （新建） | 预约中 | CreatedAt = 当前时间 | CreateReservationAsync |
| 学生取消预约 | 预约记录（Reservation） | 预约中 | 已取消 | CancelledAt = 当前时间 | CancelMyReservationAsync |

### 7.2 状态一致性规则

| 规则 | 说明 | 违反后果 |
|------|------|----------|
| 新建预约不可已有 Status | 写入时直接设 Status="预约中"，Status 字段没有默认值以外的初始值 | — |
| 已取消不可恢复 | 如果 Status="已取消" 则 Service 返回错误，不可再改为"预约中" | 逻辑混乱 |
| 同时写入 CancelledAt | 每次 Status→已取消 必须同时写入 CancelledAt = DateTime.UtcNow | 不知道取消时间 |
| 座位状态不持久化 | 座位是"空闲"还是"已预约"始终实时查询，不依赖缓存或字段 | 显示过期状态 |
| 取消操作幂等 | 对同一预约多次点击"取消"，第一次成功后后续返回 "该预约已取消，无需重复操作"，不抛异常 | 用户体验好 |

### 7.3 不可逆状态保护

在 `CancelMyReservationAsync` 和 `CancelReservationAsAdminAsync` 中均检查：

```csharp
if (reservation.Status != "预约中")
    return (false, "该预约已取消，无需重复操作");
```

这意味着：
- 已取消的记录不可恢复
- 重复取消请求安全
- Controller 拿到 `(false, msg)` 后通过 `TempData` 展示给用户

### 8.3 并发场景与防护

**场景：** 两个学生几乎同时提交同一座位的同一时段预约。

**防护策略：**
1. **第一道防线**（应用层）：`AnyAsync` 冲突检测
2. **第二道防线**（数据库层）：`IX_Reservation_Seat_Time` 唯一索引

第二道防线说明：虽然 08-7.3 指出唯一索引无法防御时段重叠（如 09:00~12:00 vs 10:00~11:00），但对于完全相同的 (SeatId, ReservationDate, StartTime, EndTime) 组合，唯一索引可以防止重复插入。在多线程极端并发下，如果应用层的 AnyAsync 恰好都返回 false，唯一索引会抛出 `DbUpdateException`，在 Service 中被捕获并转为冲突错误。

```csharp
try
{
    await _context.SaveChangesAsync();
    return (true, null);
}
catch (DbUpdateException)
{
    return (false, "该时段已被预约，请选择其他时段");
}
```

```csharp
[HttpPost]
[ValidateAntiForgeryToken]
public async Task<IActionResult> Login(AdminLoginViewModel model)
{
    if (!ModelState.IsValid)
        return View(model);

    var isValid = await _adminService.ValidateLoginAsync(model.Username, model.Password);

    if (!isValid)
    {
        ModelState.AddModelError("", "账号或密码不正确");
        return View(model);
    }

    // 创建 ClaimsPrincipal，使用 Cookie Authentication
    var claims = new List<Claim>
    {
        new Claim(ClaimTypes.Name, model.Username),
        new Claim(ClaimTypes.Role, "Admin")
    };
    var claimsIdentity = new ClaimsIdentity(claims, CookieAuthenticationDefaults.AuthenticationScheme);
    await HttpContext.SignInAsync(new ClaimsPrincipal(claimsIdentity));

    return RedirectToAction("Reservations");
}
```

```csharp
public async Task<AdminStatisticsViewModel> GetStatisticsAsync()
{
    var today = DateTime.Today;

    var totalSeats = await _context.Seats.CountAsync(s => s.IsActive);

    var todayActiveReservations = await _context.Reservations
        .Where(r => r.ReservationDate == today && r.Status == "预约中")
        .ToListAsync();

    var todayCount = todayActiveReservations.Count;
    var utilizedSeatCount = todayActiveReservations.Select(r => r.SeatId).Distinct().Count();
    var utilizationRate = totalSeats > 0
        ? Math.Round((double)utilizedSeatCount / totalSeats * 100, 0)
        : 0;

    var hotArea = await _context.Reservations
        .Where(r => r.ReservationDate == today && r.Status == "预约中")
        .Join(_context.Seats, r => r.SeatId, s => s.Id, (r, s) => s.Area)
        .GroupBy(a => a)
        .OrderByDescending(g => g.Count())
        .Select(g => g.Key)
        .FirstOrDefaultAsync() ?? "暂无数据";

    return new AdminStatisticsViewModel
    {
        TotalSeats = totalSeats,
        TodayReservations = todayCount,
        UtilizationRate = utilizationRate,
        HotArea = hotArea
    };
}
```
