# 系统设计说明 · 图书馆座位预约系统

---

## 1. 系统设计目标

| 目标 | 说明 |
|------|------|
| 开箱即用 | 克隆仓库 → dotnet run → 浏览器打开即可使用，种子数据自动初始化 |
| 单人可交付 | 代码量适中，4 周内 1 人可完成，不引入分布式、消息队列、Redis 等中间件 |
| 演示闭环 | 学生端：选座 → 预约 → 查看 → 取消；管理端：登录 → 管理 → 统计 |
| 原型可复用 | 9 个静态原型页面直接作为 Razor 视图的 HTML 骨架，复用布局和组件类 |
| 责任清晰 | Controller 只做参数验证和视图调度，Service 做业务逻辑，DataAccess 做数据操作 |

## 4. 分层说明

### 4.1 各层职责 — 一句话版

| 层 | 一句话职责 |
|----|-----------|
| **Entity** | 数据库表的 C# 映射，只含字段和关系注解，不含业务方法 |
| **ViewModel** | 传给视图的数据包装，含字段显示名、验证注解、下拉选项等视图辅助数据 |
| **Service** | 所有业务逻辑放这里（冲突检测、状态计算、统计聚合），Controller 不写业务 |
| **Controller** | 接参数 → 调 Service → 塞 ViewModel → 返回 View |重定向 |
| **DataAccess** | EF Core DbContext + SeedData，不写业务逻辑，不写 SQL |
| **View** | 从 ViewModel 渲染 HTML，只写循环/条件/表单，不写业务逻辑 |

### 4.2 Entity 层 — 数据实体

每个实体对应一张表，放在 `Models/Entities/` 下。

**Seat.cs：**
```csharp
public class Seat {
    public int Id { get; set; }
    public string SeatNumber { get; set; }      // 如 "A-12"
    public string Area { get; set; }             // 如 "三楼自习区"
    public string? Floor { get; set; }           // 如 "3F"
    public string? Description { get; set; }     // 如 "靠窗、有插座"
    public bool IsActive { get; set; } = true;
    public ICollection<Reservation> Reservations { get; set; }
}
```

**Reservation.cs：**
```csharp
public class Reservation {
    public int Id { get; set; }
    public int SeatId { get; set; }
    public string StudentName { get; set; }         // 如 "学生 A"
    public string StudentIdentifier { get; set; }   // 如 "student_a"
    public DateTime ReservationDate { get; set; }   // 只存日期部分
    public DateTime StartTime { get; set; }
    public DateTime EndTime { get; set; }
    public string Status { get; set; }              // "预约中" / "已取消"
    public DateTime CreatedAt { get; set; }
    public DateTime? CancelledAt { get; set; }
    public Seat Seat { get; set; }
}
```

**Admin.cs：**
```csharp
public class Admin {
    public int Id { get; set; }
    public string Username { get; set; }
    public string PasswordHash { get; set; }        // 哈希值，非明文
}
```

### 4.3 ViewModel 层 — 视图模型

每页对应一个 ViewModel。关键设计要点：

| ViewModel | 对应页面 | 关键字段 |
|-----------|----------|----------|
| `HomeIndexViewModel` | P-U01 首页 | `TotalSeats`, `AvailableSeats`, `OccupiedSeats`, `IsLoggedIn`, `CurrentStudentName` |
| `SeatListViewModel` | P-U02 座位列表 | `List<SeatItem>`（每个含 SeatNumber, Area, Status）, `AreaOptions`（SelectList） |
| `SeatDetailViewModel` | P-U03 座位详情 | `SeatNumber`, `Area`, `Floor`, `Description`, `IsAvailable` |
| `ReservationCreateViewModel` | P-U04 预约提交 | `SeatId`, `SeatNumber`（只读）, `ReservationDate`, `StartTime`, `EndTime`, 验证注解 |
| `ReservationListViewModel` | P-U05 我的预约 | `List<ReservationItem>`（每个含 SeatNumber, Area, Date, TimeSlot, Status, CanCancel） |
| `AdminLoginViewModel` | P-A01 登录 | `Username`, `Password` |
| `AdminReservationsViewModel` | P-A02 预约管理 | `List<ReservationItem>`, `StatusFilter`（SelectList） |
| `AdminSeatsViewModel` | P-A03 座位管理 | `List<SeatItem>`（含 Status）, `AreaOptions` |
| `AdminStatisticsViewModel` | P-A04 统计 | `TotalSeats`, `TodayReservations`, `UtilizationRate`, `HotArea` |

**ViewModel 命名约定**：每个 ViewModel 放在 `Models/ViewModels/` 下，只含 `public` 属性和 `[Required]`/`[Display]` 注解，不含方法。

### 4.4 Service 层 — 业务服务

所有业务逻辑集中在 Service 中。Controller 不得包含业务判断（除参数格式校验外）。

**ISeatService / SeatService：**

| 方法 | 输入 | 输出 | 职责 |
|------|------|------|------|
| `GetHomeStatsAsync()` | 无 | `(total, available, occupied)` | 首页 3 个统计数字 |
| `GetAllSeatsAsync(string? areaFilter)` | 区域筛选条件 | `List<SeatItem>` | 座位列表，实时计算状态 |
| `GetSeatDetailAsync(int seatId)` | 座位 ID | `SeatDetailViewModel` | 单个座位详情 + 状态 |
| `GetAllSeatsForAdminAsync()` | 无 | `List<SeatItem>` | 管理端座位列表（含停用） |

**IReservationService / ReservationService：**

| 方法 | 输入 | 输出 | 职责 |
|------|------|------|------|
| `CreateReservationAsync(...)` | seatId, student, date, start, end | `(bool success, string errorMsg)` | 冲突检测 + 创建预约 |
| `GetMyReservationsAsync(studentIdentifier)` | studentIdentifier | `List<ReservationItem>` | 查询某个学生的所有预约 |
| `CancelMyReservationAsync(int reservationId, studentIdentifier)` | 预约 ID + 学生标识 | `(bool success, string errorMsg)` | 学生取消预约（校验归属） |
| `GetAllReservationsAsync(string? statusFilter)` | 状态筛选 | `List<ReservationItem>` | 管理员查看全部预约 |
| `CancelReservationAsAdminAsync(int reservationId)` | 预约 ID | `(bool success, string errorMsg)` | 管理员取消任意预约 |

**IAdminService / AdminService：**

| 方法 | 输入 | 输出 | 职责 |
|------|------|------|------|
| `ValidateLoginAsync(string username, string password)` | 用户名 + 明文密码 | `bool` | 验证管理员账号密码 |
| `GetStatisticsAsync()` | 无 | `AdminStatisticsViewModel` | 聚合 4 个统计数字 |

**IStudentContext / StudentContext 服务：**
此服务封装 Session 操作，而非在 Controller 中直接读写 `HttpContext.Session`。

```csharp
public interface IStudentContext {
    string? CurrentStudentName { get; }         // 取当前学生名，null 表示未登录
    string? CurrentStudentIdentifier { get; }   // 取当前学生标识
    void Login(string name, string identifier); // 写入 Session
    void Logout();                               // 清除 Session
    bool IsLoggedIn { get; }
}
```

### 4.5 Controller 层 — 控制器

Controller 的代码模式：

```
public IActionResult ActionName(...) {
    // 1. 参数校验（ModelState.IsValid / 空值检查）
    // 2. 调用 Service 方法
    // 3. 构建 ViewModel
    // 4. 根据结果返回 View() 或 RedirectToAction()
}
```

每个 Action 不超过 15 行。超过说明业务逻辑没有下沉到 Service。

### 4.6 DataAccess 层

**AppDbContext.cs：**
- 继承 DbContext
- 包含 `DbSet<Seat>`、`DbSet<Reservation>`、`DbSet<Admin>`
- 在 `OnModelCreating` 中配置：唯一索引（SeatId, ReservationDate, StartTime, EndTime）、默认值、关系

**SeedData.cs：**
- `static async Task Initialize(IServiceProvider serviceProvider)`
- 检查数据库是否已初始化（Seats.Any()）
- 写入 30~50 个座位，3~4 个区域
- 写入 1 个管理员账号（密码哈希）
- 写入 1~2 条演示用预约记录

### 4.7 View 层 — 视图

Razor 视图复用原型的 HTML 结构，key 变更：

| 原型文件 | 对应 View | 主要变更 |
|----------|-----------|----------|
| `index.html` | `Home/Index.cshtml` | 硬编码数字 → `@Model.TotalSeats`，体验按钮 → `<form>` POST |
| `seats.html` | `Seats/Index.cshtml` | JS mock 数据 → `@foreach` 循环 `@Model.Seats` |
| `seat-detail.html` | `Seats/Details.cshtml` | mock 变量 → `@Model.IsAvailable` |
| `admin-login.html` | `Admin/Login.cshtml` | JS 校验 → `ModelState.IsValid` + `asp-validation-summary` |
| `admin-reservations.html` | `Admin/Reservations.cshtml` | 硬编码行 → `@foreach` + `asp-route-id` |

_Layout.cshtml 需要处理两套导航：
- 无角色 / 学生角色 → 显示"首页 / 选座 / 我的预约 / 退出"
- 管理员角色 → 显示"预约管理 / 座位管理 / 统计 / 退出"

## 8. 异常处理原则

### 8.1 异常分层处理

| 异常类型 | 处理位置 | 处理方式 |
|----------|----------|----------|
| 参数格式错误（日期格式、数字范围） | Controller | `ModelState.AddModelError()` + 返回当前 View，展示 `asp-validation-summary` |
| 业务逻辑异常（冲突、归属校验） | Service | 返回 `(false, errorMsg)` 元组，不抛异常。Controller 检查返回结果后将 errorMsg 传给 View |
| 数据库异常（连接失败、死锁） | Service/DataAccess | 全局 Exception Filter 或 `try-catch` + `ILogger` 记录，返回通用错误页 |
| 404（座位不存在、预约不存在） | Controller | `return NotFound()` 或返回友好页（Error.cshtml） |
| 未授权访问（未登录进管理端） | Controller | `return RedirectToAction("Login")` |

### 8.2 业务错误返回约定

所有 Service 方法遇到可预见的业务错误（冲突、无权操作、重复操作）均使用**返回元组模式**：

```csharp
public async Task<(bool Success, string ErrorMessage)> CreateReservationAsync(
    int seatId, string studentName, string studentIdentifier,
    DateTime date, DateTime start, DateTime end)
{
    // 1. 查询已有预约判断冲突
    // 2. 冲突 → return (false, "该时段已被预约，请选择其他时段")
    // 3. 正常 → 写入数据库 → return (true, null)
}
```

禁止在 Service 中为业务错误抛 `Exception`。仅有不可恢复错误（数据库连接失败等）才抛异常。

### 8.3 View 中的错误展示

| 错误类型 | 展示方式 |
|----------|----------|
| 表单字段验证 | `<span asp-validation-for="FieldName">` 红色字段下方文字 |
| 业务逻辑错误 | `ModelState.AddModelError("", msg)` → `asp-validation-summary` 顶部红色提示框 |
| 无数据/空状态 | View 中 `@if (!Model.Items.Any())` → 空状态卡片 + 引导按钮 |
| 加载失败 | 本项目纯服务端渲染，无前端 AJAX 加载失败场景。若需模拟可添加 `try-catch` + ViewBag 错误标记 |

## 9. Session / Cookie 使用原则

### 9.1 学生身份（Session，轻量级）

```csharp
// StudentContext 封装 Session 操作
public class StudentContext : IStudentContext {
    private readonly IHttpContextAccessor _httpContextAccessor;

    public string? CurrentStudentName =>
        _httpContextAccessor.HttpContext?.Session.GetString("StudentName");

    public string? CurrentStudentIdentifier =>
        _httpContextAccessor.HttpContext?.Session.GetString("StudentIdentifier");

    public void Login(string name, string identifier) {
        _httpContextAccessor.HttpContext?.Session.SetString("StudentName", name);
        _httpContextAccessor.HttpContext?.Session.SetString("StudentIdentifier", identifier);
    }

    public void Logout() {
        _httpContextAccessor.HttpContext?.Session.Remove("StudentName");
        _httpContextAccessor.HttpContext?.Session.Remove("StudentIdentifier");
    }
}
```

**Session 内容**：仅存 2 个 string（名称 + 标识符），约 50 字节。无过期时间（浏览器进程级别）。

**访问方式**：Controller 中注入 `IStudentContext` 接口，通过 `.CurrentStudentName` / `.CurrentStudentIdentifier` 读取。禁止直接使用 `HttpContext.Session["..."]`。

### 9.2 管理员身份（Cookie，临时认证）

管理员使用 ASP.NET Core 内置的 Cookie Authentication：

```csharp
// Program.cs
builder.Services.AddAuthentication(CookieAuthenticationDefaults.AuthenticationScheme)
    .AddCookie(options => {
        options.LoginPath = "/Admin/Login";
        options.LogoutPath = "/Admin/Logout";
        options.AccessDeniedPath = "/Admin/Login";
    });
```

登录成功时：`HttpContext.SignInAsync(principal, properties)`。
退出时：`HttpContext.SignOutAsync()`。

管理员 Cookie 生命周期：会话级别（关闭浏览器即失效）。

### 9.3 为什么不直接用 Identity

ASP.NET Core Identity 功能强大但过于重型——它包含用户注册、密码重置、角色管理、邮箱确认等整套机制，而本项目只需要 1 个预置管理员账号。手写 Cookie 认证 + Session 学生身份，总代码量不到 50 行。

### 9.4 权限拦截策略

| 被保护的资源 | 拦截方式 | 未登录时行为 |
|-------------|----------|-------------|
| `/Reservations/*`（学生端预约页面） | Controller 中检查 `StudentContext.IsLoggedIn`，如 false 则重定向到首页 | `return RedirectToAction("Index", "Home")` |
| `/Admin/*`（管理端所有页面） | `[Authorize]` Attribute | 自动跳转 `/Admin/Login` |
| 学生端首页的统计数据 | 根据 `IsLoggedIn` 显示/隐藏体验按钮或欢迎语 | 显示体验按钮 + 不显示预约数据 |

## 附录 A：DbContext OnModelCreating 配置要点

```csharp
protected override void OnModelCreating(ModelBuilder modelBuilder)
{
    // Seat
    modelBuilder.Entity<Seat>(entity => {
        entity.HasIndex(s => s.SeatNumber).IsUnique();
        entity.Property(s => s.SeatNumber).HasMaxLength(20).IsRequired();
        entity.Property(s => s.Area).HasMaxLength(50).IsRequired();
        entity.Property(s => s.Floor).HasMaxLength(20);
        entity.Property(s => s.Description).HasMaxLength(200);
    });

    // Reservation
    modelBuilder.Entity<Reservation>(entity => {
        entity.HasIndex(r => new { r.SeatId, r.ReservationDate, r.StartTime, r.EndTime })
              .HasDatabaseName("IX_Reservation_Seat_Time");

        entity.Property(r => r.StudentName).HasMaxLength(50).IsRequired();
        entity.Property(r => r.StudentIdentifier).HasMaxLength(50).IsRequired();
        entity.Property(r => r.Status).HasMaxLength(20).IsRequired();
        entity.Property(r => r.ReservationDate).HasColumnType("date");

        entity.HasOne(r => r.Seat)
              .WithMany(s => s.Reservations)
              .HasForeignKey(r => r.SeatId)
              .OnDelete(DeleteBehavior.Restrict); // 不允许级联删除
    });

    // Admin
    modelBuilder.Entity<Admin>(entity => {
        entity.HasIndex(a => a.Username).IsUnique();
        entity.Property(a => a.Username).HasMaxLength(50).IsRequired();
        entity.Property(a => a.PasswordHash).HasMaxLength(200).IsRequired();
    });
}
```

## 附录 B：冲突检测 SQL 等价逻辑

ReservationService 中的冲突检测等价于以下 SQL：

```sql
SELECT COUNT(*) FROM Reservations
WHERE SeatId = @seatId
  AND ReservationDate = @date
  AND Status = '预约中'
  AND StartTime < @endTime
  AND EndTime > @startTime
```

如果 COUNT > 0，则冲突。对应的 LINQ：

```csharp
var conflict = await _context.Reservations.AnyAsync(r =>
    r.SeatId == seatId &&
    r.ReservationDate.Date == date.Date &&
    r.Status == "预约中" &&
    r.StartTime < endTime &&
    r.EndTime > startTime);
```

// 在 AdminService.GetStatisticsAsync() 中
var today = DateTime.Today;
var totalSeats = await _context.Seats.CountAsync(s => s.IsActive);

    .Where(r => r.ReservationDate == today && r.Status == "预约中")

var utilizationRate = totalSeats > 0
    ? Math.Round((double)utilizedSeats / totalSeats * 100, 0)
    : 0;
var hotArea = await _context.Reservations
    .Where(r => r.ReservationDate == today && r.Status == "预约中")
    .GroupBy(r => r.Seat.Area)
    .OrderByDescending(g => g.Count())
    .Select(g => g.Key)
    .FirstOrDefaultAsync() ?? "暂无数据";
