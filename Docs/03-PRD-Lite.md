# PRD Lite · 图书馆座位预约系统

---

## 1. 项目概述

| 项目 | 内容 |
|------|------|
| 项目名称 | 图书馆座位预约系统 |
| 技术栈 | ASP.NET Core MVC (.NET 8) + Razor + SQL Server LocalDB + EF Core + Bootstrap 5 |
| 开发周期 | 4 周（课程第三阶段） |
| 开发人数 | 1 人 |
| 交付方式 | GitHub / Gitee 仓库链接 |

**一句话描述**：开发一个 Web 系统，让学生能在线查看空闲座位、预约座位、取消预约，让管理员能管理预约、查看基本统计。

---

## 2. 项目目标

1. 覆盖"选座 → 预约 → 查看 → 取消"的学生端核心闭环
2. 覆盖"登录 → 管理预约 → 查看统计"的管理端核心闭环
3. 系统开箱即用：启动时自动初始化种子数据（30-50 个座位 + 1 个管理员账号 + 3 个体验学生身份）
4. 最终提交完整可运行的 Git 仓库，包含 README 运行说明

---

## 3. 目标用户

| 角色 | 身份 | 使用方式 | 核心诉求 |
|------|------|----------|----------|
| 学生 | 在校本科生 | 浏览器打开系统，点击体验账号直接使用 | 快速找空位、预约、取消 |
| 管理员 | 图书馆工作人员 | 输入账号密码登录管理后台 | 管理预约、看统计数据 |

**本期不开发注册流程**：学生端预置 3 个体验账号（学生 A/B/C），点击即切换身份；管理端预置 admin/123456。

---

## 4. 使用场景

| 场景 | 用户 | 触发条件 | 目标 |
|------|------|----------|------|
| S-01 考前预约座位 | 学生 | 计划去图书馆学习 | 提前锁定座位，到馆直接坐 |
| S-02 临时到馆找座 | 学生 | 已到图书馆，找空位 | 快速找到并预约空闲座位 |
| S-03 取消预约 | 学生 | 计划改变，座位用不上 | 释放座位 |
| S-04 查看自己的预约 | 学生 | 忘了约了哪个座位、什么时段 | 确认预约信息 |
| S-05 管理员查看预约情况 | 管理员 | 日常巡查或处理投诉 | 了解所有预约 |
| S-06 管理员取消违规预约 | 管理员 | 学生预约后未到场 | 释放座位 |
| S-07 管理员查看统计 | 管理员 | 期末汇报或评估利用率 | 获取座位使用数据 |

## 5. 用户端核心功能

### F1 体验账号登录

| 项目 | 内容 |
|------|------|
| 入口 | 系统首页，3 个按钮："体验学生 A""体验学生 B""体验学生 C" |
| 逻辑 | 点击后写 Cookie/Session 标记当前用户身份，跳转首页 |
| 切换 | 再次点击其他体验账号即可切换身份 |
| 退出 | 提供"退出"按钮，清除身份标记，回到未登录首页 |

**最小验收标准**：
- AC1.1：未登录时首页显示 3 个体验按钮，不显示任何预约数据
- AC1.2：点击"体验学生 A"后，页面顶部显示"欢迎，学生 A"
- AC1.3：切换到"体验学生 B"，预约列表中学生 B 看不到学生 A 的预约数据

### F2 空闲座位概览

| 项目 | 内容 |
|------|------|
| 位置 | 用户首页顶部卡片 |
| 数据 | 总座位数、当前空闲座位数、已预约座位数 |

**最小验收标准**：
- AC2.1：数字与座位列表页实际数据一致
- AC2.2：有学生预约或取消后，刷新页面数字同步更新

### F3 座位列表

| 项目 | 内容 |
|------|------|
| 展示 | 卡片或表格，每行显示座位编号、区域、状态 |
| 状态颜色 | 空闲=绿色、已预约=橙色 |
| 筛选 | 下拉框按区域筛选（一楼大厅、二楼阅览区、三楼自习区等） |
| 详情入口 | 每个座位有"查看"按钮，点击弹出详情 Modal |

**最小验收标准**：
- AC3.1：列表显示全部座位，空闲与已预约颜色不同
- AC3.2：选择区域筛选后，列表只显示该区域座位
- AC3.3：点击"查看"弹出 Modal，显示座位编号、区域、楼层、设施描述

### F4 预约提交

| 项目 | 内容 |
|------|------|
| 入口 | 座位详情 Modal 中的"立即预约"按钮 |
| 表单 | 座位号（只读）、日期选择、开始时间、结束时间 |
| 冲突检测 | 提交时检查该时段该座位是否已被预约，冲突则提示错误 |
| 成功 | 页面提示"预约成功"，跳转到"我的预约" |
| 失败 | 提示冲突详情，不跳转 |

**最小验收标准**：
- AC4.1：选择空闲座位+未来时段，提交成功，生成状态为"预约中"的记录
- AC4.2：选择已被他人预约的时段，提交失败，提示"该时段已被预约"
- AC4.3：选择开始时间晚于结束时间，提交失败，提示"结束时间不能早于开始时间"
- AC4.4：选择过去的时间，提交失败，提示"不能预约过去的时间"
- AC4.5：同一学生不能在同一时段预约两个不同座位

### F5 我的预约

| 项目 | 内容 |
|------|------|
| 展示 | 表格列表，显示座位号、区域、时段、状态（预约中/已取消） |
| 取消操作 | 每行"预约中"记录有"取消"按钮 |
| 排序 | 按预约创建时间倒序 |

**最小验收标准**：
- AC5.1：列表只显示当前学生的预约
- AC5.2：预约成功的记录出现在列表中，状态为"预约中"
- AC5.3：取消预约后，状态变为"已取消"，座位列表中该座位恢复空闲

### F6 取消预约

| 项目 | 内容 |
|------|------|
| 位置 | 我的预约页面，每条"预约中"记录的"取消"按钮 |
| 确认 | 点击后弹出确认对话框（"确定取消该预约？"） |
| 结果 | 状态变为"已取消"，座位释放 |
| 限制 | 只能取消自己的预约；已取消的记录不可再取消 |

**最小验收标准**：
- AC6.1：取消后我的预约中该记录状态变为"已取消"
- AC6.2：取消后座位列表中该座位状态恢复为"空闲"
- AC6.3：其他学生可以看到该座位并可预约

## 6. 管理端核心功能

### F7 管理员登录

| 项目 | 内容 |
|------|------|
| 页面 | 独立的登录页，账号和密码输入框 |
| 认证方式 | Session / Cookie 认证 |
| 默认账号 | admin / 123456 |
| 失败处理 | 账号或密码错误时提示"账号或密码不正确" |

**最小验收标准**：
- AC7.1：输入正确账号密码，登录成功，跳转到预约管理页
- AC7.2：输入错误密码，提示错误，不跳转
- AC7.3：未登录直接访问管理端页面，跳转到登录页

### F8 预约管理

| 项目 | 内容 |
|------|------|
| 展示 | 全部学生的预约记录列表 |
| 筛选 | 按状态（预约中/已取消）筛选 |
| 取消操作 | 管理员可取消任意"预约中"的记录 |

**最小验收标准**：
- AC8.1：列表显示所有学生的预约记录
- AC8.2：筛选"预约中"后只显示状态为"预约中"的记录
- AC8.3：管理员取消预约后，该记录状态变为"已取消"，座位释放

### F9 统计页

| 项目 | 内容 |
|------|------|
| 展示 | 4 个数字卡片：总座位数、今日预约数、座位利用率(%)、热门区域名称 |
| 数据源 | 从数据库实时查询 |

**最小验收标准**：
- AC9.1：总座位数与种子数据一致
- AC9.2：今日预约数为当天提交的预约总数
- AC9.3：座位利用率 = 当天有预约的座位数 / 总座位数 x 100%
- AC9.4：热门区域显示预约次数最多的区域名称

## 8. 核心业务流程

### 学生主流程（选座 —> 预约 —> 查看 —> 取消）

`
开始
  |
  +-- [未登录] 首页 --> 点击"体验学生A/B/C"
  |               --> 写入 Session/Cookie --> 转登录后首页
  |
  +-- [已登录] 首页（看到空闲座位数）
  |               --> 点击"选座预约"
  |               --> 座位列表页（看到所有座位，可筛选区域）
  |                     |
  |                     +-- 点击空闲座位"查看" --> 弹出详情 Modal
  |                     |     --> 点击"立即预约"
  |                     |     --> 填写日期+时段 --> 提交
  |                     |         +-- [冲突或无异常] --> 预约成功 --> 跳转我的预约
  |                     |         +-- [冲突] --> 提示错误，不跳转
  |                     |
  |                     +-- 点击"我的预约"
  |                           --> 看到所有预约记录
  |                                 +-- 点击"取消" --> 确认 --> 状态变已取消
  |
  +-- [退出] 点击退出 --> 清除身份 --> 回到首页
`

### 管理员流程（登录 --> 管理 --> 统计）

`
开始
  |
  +-- [未登录] 访问 /Admin/* --> 跳转管理员登录页
  |               --> 输入 admin/123456
  |               --> 登录成功 --> 预约管理页
  |
  +-- [预约管理] 查看全部预约
  |               +-- 按状态筛选
  |               +-- 点击"取消" --> 确认 --> 释放座位
  |
  +-- [统计页] 查看 4 个数字卡片
  |
  +-- [退出] 点击退出 --> 清除 Session --> 回到学生端首页
`

## 11. 核心数据对象

### 11.1 数据表关系

系统共 3 张数据表：

`
体验学生（内存中模拟，不建表）
    |
    | 1
    | 拥有 N 条
    |
    v
Reservations（预约记录表）—— N:1 —— Seats（座位表）

Admin（管理员表，仅种子数据写入）
`

### 11.2 Seats（座位表）

| 字段 | 类型 | 说明 |
|------|------|------|
| Id | int (PK) | 主键，自增 |
| SeatNumber | string(20) | 座位编号，如 "A-12" |
| Area | string(50) | 区域名称，如 "三楼自习区" |
| Floor | string(20) | 楼层，如 "3F" |
| Description | string(200) | 设施描述，如 "靠窗、有插座" |
| IsActive | bool | 是否启用，默认 true |

**种子数据**：30-50 条，覆盖 3-4 个区域，每个区域 8-15 个座位。

### 11.3 Reservations（预约记录表）

| 字段 | 类型 | 说明 |
|------|------|------|
| Id | int (PK) | 主键，自增 |
| SeatId | int (FK) | 关联 Seats 表 |
| StudentName | string(50) | 学生名称，如 "学生 A" |
| StudentIdentifier | string(50) | 学生标识，如 "student_a" |
| ReservationDate | DateTime | 预约日期（只存日期部分） |
| StartTime | DateTime | 预约开始时间 |
| EndTime | DateTime | 预约结束时间 |
| Status | string(20) | 状态：预约中 / 已取消 |
| CreatedAt | DateTime | 创建时间 |
| CancelledAt | DateTime? | 取消时间（可为空） |

**索引**：(SeatId, ReservationDate, StartTime, EndTime) 唯一组合索引，防止同一座位同一时段重复预约。

### 11.4 Admin（管理员表）

| 字段 | 类型 | 说明 |
|------|------|------|
| Id | int (PK) | 主键，自增 |
| Username | string(50) | 用户名，种子写入 "admin" |
| PasswordHash | string(200) | 密码哈希（非明文存储） |

**种子数据**：1 条，admin / 123456（实际存哈希值）。

### 11.5 关于体验学生不建表的说明

3 个体验学生（A/B/C）不在数据库中建表，也不存在真实的用户注册流程。他们的身份仅通过前端 Cookie/Session 标记区分，用于演示"不同学生看到不同的预约数据"。这是一个刻意的简化决策，由需求分析阶段确认。

## 12. 状态流转说明

### 12.1 座位状态

座位只有 2 种状态（实时计算，不存字段）：

`
空闲 <——> 已预约
`

- 当一个座位在当前时段没有任何"预约中"的记录时，状态为"空闲"
- 当该座位在当前时段至少有一条"预约中"的记录时，状态为"已预约"
- 座位状态通过查询 Reservations 表实时计算，不额外存储

### 12.2 预约记录状态

预约记录有 2 种状态（存储在 Status 字段）：

`
         +-- 取消 --> 已取消
         |
 预约中 --+
         |
         +-- 时间到（暂不处理，V1.1 支持自动过期）
`

- **预约中**：学生提交预约后的初始状态
- **已取消**：学生或管理员手动取消后的最终状态
- 预约记录不支持从"已取消"恢复到"预约中"
- V1.0 不实现自动过期/签到，预约到期后需管理员手动取消或等待 V1.1 功能

### 12.3 冲突检测规则

冲突检测在提交预约时触发，规则如下：

`
同一座位  +  同一天  +  时段重叠（StartTime1 < EndTime2 AND StartTime2 < EndTime1）
          =  冲突，拒绝提交
`

示例：
- 座位 A-01 已有预约 09:00-12:00
- 新预约 10:00-11:00 --> 冲突（重叠）
- 新预约 08:00-09:00 --> 不冲突（刚好衔接）
- 新预约 12:00-14:00 --> 不冲突（刚好衔接）
