## C. 必改项（P0，阻塞系统设计）

以下 4 项必须在静态原型阶段**修复并复审通过**后方可进入系统设计说明：

| # | 问题 | 影响文件 | 严重性说明 |
|---|------|---------|-----------|
| C-1 | **管理端侧边栏在手机端不折叠** | `admin-reservations.html`, `admin-seats.html`, `admin-statistics.html` | 管理员可能用手机应急查看，当前侧边栏 220px 在 <375px 设备上占满屏幕，内容区几乎不可用 |
| C-2 | **座位列表页缺少骨架屏加载状态** | `seats.html` | 05-页面卡明确要求"加载中显示 6 个灰色骨架块"；缺失会使用户在数据返回前看到白屏或闪烁 |
| C-3 | **预约提交页缺少冲突错误提示区域（UI 预留位）** | `reservation-create.html` | 05-PRD 冲突规则要求在服务端检测重复时间段；前端必须预留错误提示区域，否则冲突发生时用户得不到反馈 |
| C-4 | **座位详情页未覆盖"已被他人预约"的展示状态** | `seat-detail.html` | 当前只展示了空闲状态的详情页；若座位已被他人预约，用户应看到不可预约的提示和可选时间段建议 |

### 必改项修复指引

| # | 修改方案 |
|---|---------|
| C-1 | 在 CSS 中添加 `@media (max-width: 767px) { .admin-sidebar { display: none; } }`，并增加汉堡菜单按钮，点击后 JS 切换侧边栏可见性（`toggle` class）。参考典型 Bootstrap 4 Navbar 折叠模式。 |
| C-2 | 在 `seats.html` 的 `#seat-list` 区域内初始渲染 6 个 `<div class="skeleton skeleton-card">`，数据加载完成后替换为实际座位卡片。CSS 中已有 `skeleton` 和 `shimmer` 动画，直接使用。 |
| C-3 | 在 `reservation-create.html` 的卡片上方（或表单区域）增加一个 `#conflict-error` 区域，初始 `display:none`，样式同 `.alert-error`。 |
| C-4 | 在 `seat-detail.html` 的 JS 中增加状态判断分支，当 `seat.status === 'occupied'` 时展示："此座位已被预约"的警示提示 + 按钮禁用或改为"查看可选座位"链接。 |
