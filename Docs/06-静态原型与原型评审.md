## 三、视觉方向与风格

### 3.1 视觉方向

**安静学习感（Quiet Study Atmosphere）**

围绕图书馆"安静、专注、秩序"的场景氛围设计。大面积留白、柔和蓝色主调、克制的灰阶层级。

### 3.2 用户端风格

| 维度 | 策略 |
|------|------|
| 布局 | 手机优先（mobile-first），单列卡片流式布局 |
| 导航 | 顶部 60px 白底导航栏 + 三链接（首页/选座/我的预约） |
| 卡片 | 白色背景、4px 圆角、1px 阴影、20px 内边距 |
| 内容区 | 单列满宽，`<768px` 不切换 |
| 最大宽度 | `<768px` 满宽，`≥768px` Bootstrap container 自适应 |
| 操作按钮 | 满宽底部放置或 flex-fill 双按钮 |
| 底部安全区 | 32px 留白 |

### 3.3 管理端风格

| 维度 | 策略 |
|------|------|
| 布局 | 桌面优先，顶部导航栏 + 左侧 220px 侧边栏 + 右侧内容区 |
| 侧边栏 | 白底、右边框、48px 行高、激活态 3px 蓝色左边框 + 浅蓝底 |
| 表格 | 桌面全宽展示，`<768px` 横向滚动 |
| 手机适配 | 汉堡菜单折叠侧边栏 + 遮罩层 |
| 登录页 | 居中窄卡 400px，全屏居中 |

|------|----------|------|

22 个样式模块，涵盖：CSS 变量体系、导航栏、卡片、统计卡片、状态标签、按钮、表单、表格、后台布局、座位卡片、骨架屏动画、警告提示、空状态、登录页、弹窗、响应式侧边栏折叠等。

|------|------|------|

| 管理预约数据 | `admin-reservations.html` 硬编码 | 4 条记录（3 种状态），3 个模拟学生 |
| 管理座位数据 | `admin-seats.html` 硬编码 | 5 条记录，3 个区域 |

**数据特性：** 纯静态模拟，页面刷新重置，无持久化。交互操作（取消预约/删除座位/登录）仅改变当前 DOM 状态，不会跨页面同步。

## 五、未接入逻辑说明

| 逻辑 | 说明 |
|------|------|
| 真实后端 API | 无，所有数据为前端硬编码或 JS 数组 |
| 数据库持久化 | 无，数据在页面刷新后重置 |
| 用户认证 | 无真实 Session/Cookie，管理员登录本地校验 `admin/123456` |
| 冲突检测 | 无服务端检测，`reservation-create.html` 中可用 JS 变量 `hasConflict` 模拟 |
| 时间段校验 | 仅前端时间大小校验，无后端时段冲突逻辑 |
| 数据同步 | 取消预约在管理端和用户端各自独立，不会同步 |
