## C. 逐项复核

### C1 复核：P0-01 IX_Reservation_Seat_Time 索引唯一性

**审计主张：** D03(唯一)→D07(默认非唯一)→D08(非唯一)→D09(唯一) 四份文档不一致。

**源文档验证：**

| 文档 | 实际内容 | 验证 |
|------|----------|------|
| D03-11.3 | "建立唯一组合索引(SeatId, ReservationDate, StartTime, EndTime)防止重复预约" | ✅ 主张唯一 |
| D07-附录A | `HasIndex(r => new { r.SeatId, r.ReservationDate, r.StartTime, r.EndTime }).HasDatabaseName("IX_Reservation_Seat_Time")` 无 `.IsUnique()` | ✅ 默认非唯一 |
| D08-4.2 | 索引类型列标注"**非唯一**" | ✅ 声明非唯一 |
| D09-8.3 | "唯一索引 IX_Reservation_Seat_Time 作为第二道防线" + `catch (DbUpdateException)` 捕获重复键异常 | ✅ 主张唯一 |

**复核结论：✅ 确认为 P0（真阻塞）。** EF Core 配置 `.IsUnique()` 与否生成的迁移 SQL 完全不同。开发者无法从四份矛盾文档中判断正确做法。必须编码前统一。

**注意：** D08-7.3 解释了"非唯一"的理由（不能防御时段重叠），但 D09-8.3 指出唯一索引仍有价值（防完全相同的时间组合双重提交）。两者不矛盾——唯一索引不能防御重叠但能防完全重复。最终应明确：索引定义为 **UNIQUE**，同时在 D08-7.3 中补充说明"唯一索引只防精确重复，不防时段重叠"。

### C2 复核：P0-02 座位详情实现方式（Modal vs 独立页）

**审计主张：** D02节7.3/D03决定用Modal，D04起改为独立页，存在矛盾。

**关键问题：所有实现文档（D04+）是否一致？**

| 文档 | 决策 | 角色 |
|------|------|------|
| D02 MVP确认 | Modal | 早期范围决策 |
| D03 PRD | Modal | 早期范围决策 |
| **D04 页面树** | **独立页 PU-03** | **页面结构定义** |
| **D05 页面卡** | **独立页 PU-03** | **UI规范定义** |
| **D06 原型** | **独立页 seat-detail.html** | **视觉实现参考** |
| **D07 系统设计** | **独立页 Seats/Details.cshtml** | **架构实现依据** |
| **D09 关键链路** | **独立页 L1-4** | **编码实现依据** |

**复核结论：❌ 不构成 P0，应将 P0-02 降级为 P2。** 理由：
1. 所有编码实现文档（D04~D09）全部统一为独立页，开发者不会选错
2. D02/D03 是早期范围决策文档，后期设计阶段已重新决策
3. 如果开发者在 Sprint 1 实现时只看 D07/D09（系统设计和链路设计），不会遇到矛盾
4. D02/D03 未修正只是历史文档不同步，不影响任何一行代码

**建议处理：** 在 D02 或 D03 中添加脚注"注：此决策在 D04 页面树阶段修订为独立页"（约 3 分钟），或暂不处理留到编码后统一清理。

## E. Go/No-Go 结论复核

### E.1 原始结论

> **No-Go** — 必须消除全部 6 项 P0 冲突 + 清理 L01~L07 共 7 项遗留审计问题后，重新审计方可进入 Sprint 0 代码开发。

### E.2 复核结论

| 裁决 | 条件 |
|------|------|
| **⚠️ 修正为：Conditional Go** | 修复 P0-01（索引唯一性跨文档统一）+ A09 C1/C2（远端地址同步+推送）后即可进入 Sprint 0 代码开发。其余 P1/P2 项在 Sprint 0~1 迭代中同步修复。 |

### E.3 修正理由

1. **6 项 P0 中 5 项降级，剩余 1 项真 P0 影响范围有限**（只涉及 D07 一行代码和 D08 一行说明）
2. **总审计将所有跨文档不一致都升格为 P0**，但"早期文档落后于后期决策"不等于"编码时不知道怎么做"。开发者编码时以 D07/D09 为准，大部分矛盾不会进入代码
3. **No-Go 的门槛过高**会带来修复周期延长。10 项 P0 修复项（G-01~G-10）中大部分是说明性修改（改文档描述），不需要代码验证，可以并行推进
4. **D10 的 Sprint 0 任务完全独立于这些文档争议**——创建 .sln、.csproj、NuGet 包、目录结构等不受索引唯一性影响

### E.4 安全进入开发的栅栏

```
栅栏 1：修复 P0-01（索引唯一性）
   → D07 附录A: 添加 .IsUnique()
   → D08 4.2: "非唯一" → "唯一" + 7.3补充说明
   → D09 8.3: 与 D08 同步（D09 已是唯一，无需改，但加注释引用 D08）
   → A07 C1 标记为已修复
   
栅栏 2（A09 遗留）：远端推送
   → D10 1.2 更新为实际地址
   → 配置 PAT 或 SSH 完成 git push
   
栅栏 3（可选快速修复，~15 分钟）：
   → D04 添加座位管理口径声明（A04 D1）
   → D06 添加路由勘误说明
   → D03 U-05 状态修正
   
通过栅栏 → 进入 Sprint 0
余下 P1/P2 → Sprint 0~1 迭代中处理
```
