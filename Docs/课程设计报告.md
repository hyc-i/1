## 1. 功能设计

### 1.1 系统目标

学生第二课堂登记管理系统面向高校学生第二课堂学分认定场景，围绕“学生档案维护、活动材料登记、管理员审核授分、学生第二课堂信息查询”四个核心环节进行设计。系统采用 ASP.NET Core MVC 技术实现 Web 应用，使用 SQLite 数据库保存学生档案和第二课堂活动记录，首次运行自动创建数据库与示例数据，便于部署、演示与提交。

系统目标如下：

1. 完成学生基本信息的增加、删除、修改、查询与批量导入。
2. 支持学生登记本人参加的社会活动、科研与竞赛、评奖评优、认证考试、奖励表彰等第二课堂记录。
3. 支持管理员对登记记录进行审核，填写审核意见并授予第二课堂学分。
4. 支持管理员按学生查询第二课堂登记明细、待审核数量、通过数量与累计学分。
5. 提供美观、清晰、便于演示的 Web 界面，满足课程设计对功能完备、技术应用、界面设计和软件成熟度的要求。

### 2.2 其他设计

1. **索引设计：** 对 `Students.StudentNo`、`ActivityRecords.StudentId`、`ActivityRecords.Status`、`ActivityRecords.Category` 建立索引，提高学号查询、学生明细查询、状态筛选和类别筛选效率。
2. **级联删除：** `ActivityRecords.StudentId` 外键设置 `ON DELETE CASCADE`，删除学生档案时自动删除该学生的登记记录。
3. **触发器：** `trg_activity_reviewed_at` 在审核状态、学分或审核意见更新后自动维护 `ReviewedAt` 字段，保证审核时间可追溯。
4. **初始化数据：** 首次运行系统时，`DatabaseInitializer` 会执行 `Database/schema.sql` 并插入 5 条学生样例与 5 条第二课堂样例记录，便于演示系统功能。

## 4. 设计总结

本课程设计完成了学生第二课堂登记管理系统的主要功能，包括学生信息维护、学生名单导入、第二课堂活动登记、审核授分和管理员查询统计。项目采用 MVC 分层结构：模型层描述学生和活动记录，数据层负责 SQLite 初始化和 ADO.NET 操作，控制器层处理业务流程，视图层负责页面展示与交互。系统首次运行即可自动创建数据库文件，便于提交和演示。

本次开发过程中使用 AI 辅助完成了需求拆解、项目结构规划、代码生成、界面风格设计、课程文档整理和调试排错。AI 的收益主要体现在：能够快速根据任务书梳理功能模块；能生成完整的 ASP.NET Core MVC 项目代码；能帮助设计数据库表、视图和触发器；能根据评分标准优化界面美观度和文档完整性。使用 AI 过程中也遇到了一些问题，例如 Windows 与 WSL 之间中文路径编码、Windows 侧 `dotnet.exe` 启动后 WSL 访问本机回环地址不一致、浏览器截图需要等待进程结束等。这些问题需要人工判断原因并调整命令，说明 AI 在复杂本地环境和跨平台细节上仍需要人工验证。
