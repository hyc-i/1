## 3. 做得好的地方

- Service 层与 Controller 层职责清晰，Controller 保持精简

| L-01 | 日期筛选未按天限定已占用座位（5 处：GetHomeStatsAsync / GetAllSeatsAsync / GetAllSeatsForAdminAsync / GetSeatDetailAsync / GetStatisticsAsync 热门区域） | P1 | 统计跨天数据可能不准确 |

## 6. 最终演示顺序建议

```
1. dotnet run --project src/LibrarySeatReservation.Web
   → 浏览器打开 http://localhost:5193

2. 首页展示（未登录态）
   → 3 个统计数字（总座位/空闲/已预约）
   → 3 个体验按钮（学生A/B/C）

3. 点击"学生A"登录
   → 页面刷新为已登录态
   → 显示"当前身份：学生A"

4. 点击"查看座位"
   → 座位卡片列表（40 个座位，3 个区域）
   → 下拉筛选区域

5. 点击空闲座位的"预约"按钮
   → 预约表单（座位号只读、日期默认明天、时段默认 9:00-10:00）

6. 确认预约
   → 自动跳转"我的预约"
   → 看到新预约记录（状态"预约中"）

7. 点击"取消"按钮（确认弹窗）
   → 状态变为"已取消"
   → 弹出 TempData 成功消息

8. 访问 /Admin/Login
   → 输入 admin / 123456
   → 进入管理端预约管理页

9. 查看预约列表
   → 看到刚才学生A的预约/取消记录
   → 状态下拉筛选

10. 点击"座位管理"
    → 看到全部座位列表（含状态标签）
    → 可新增/编辑/停用

11. 点击"数据统计"
    → 4 个数字卡片（总座位/今日预约/利用率/热门区域）

12. 点击"返回前台"退出管理端
```
