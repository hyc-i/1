## C. 必改项

### C-1 · P0-01 唯一索引修复（最高优先级）

**位置**：`src/LibrarySeatReservation.Web/DataAccess/AppDbContext.cs:45-46`

**当前代码**：
```csharp
e.HasIndex(r => new { r.SeatId, r.ReservationDate, r.StartTime, r.EndTime })
    .HasDatabaseName("IX_Reservation_Seat_Time");
```

**应改为**：
```csharp
e.HasIndex(r => new { r.SeatId, r.ReservationDate, r.StartTime, r.EndTime })
    .IsUnique()
    .HasDatabaseName("IX_Reservation_Seat_Time");
```

**同步修复**：
1. `Docs/13-用户端主链路开发记录.md` 第 20 行和第 115 行已声称修复，需确认代码实际生效
2. `Docs/项目任务板与迭代记录.md` 一致性修订跟踪 P0-01 状态需从"⬜ 未处理"改为"🔄 已处理"或"✅ 已复核"
3. 修复后需重新生成 EF Core 迁移（`dotnet ef migrations add FixUniqueIndex`）

### C-2 · 日期筛选修复（P1-1 ~ P1-5）

在以下 4 个查询中增加 `r.ReservationDate == now.Date`（或 `== today`）条件：

1. `SeatService.cs:21-22` — `GetHomeStatsAsync`
2. `SeatService.cs:37-38` — `GetAllSeatsAsync`
3. `SeatService.cs:59-60` — `GetSeatDetailAsync`
4. `SeatService.cs:88-89` — `GetAllSeatsForAdminAsync`
5. `AdminService.cs:42-43` — `GetStatisticsAsync` 热门区域查询

### C-3 · 时区修复（P1-6 ~ P1-7）

1. `ReservationsController.cs:40`：`DateTime.UtcNow.Date.AddDays(1)` → `DateTime.Today.AddDays(1)`
2. `ReservationService.cs:28`：`DateTime.UtcNow.Date` → `DateTime.Today`

**注意**：修复时区后需确认 `ReservationDate` 字段的存储类型（`HasColumnType("date")`）与比较逻辑一致。`DateTime.Today` 返回本地时区日期，与 `date` 列类型匹配。

### C-4 · Admin/Reservations 视图修复（P1-8）

需三处同步修改：

1. **`Models/ViewModels/ReservationItem.cs`**：添加 `public string StudentName { get; set; } = string.Empty;`
2. **`Services/ReservationService.cs` `GetAllReservationsAsync`**：在 `Select` 中添加 `StudentName = r.StudentName`
3. **`Views/Admin/Reservations.cshtml`**：在 `<tbody>` 的 `<tr>` 中、`<td>@item.Area</td>` 之后添加 `<td>@item.StudentName</td>`
