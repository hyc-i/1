# 页面卡与 UI 规范 · 审计报告

审计对象：Docs/05-页面卡与UI规范.md  
审计日期：2026-07-07  
审计结论：**有条件通过**，存在 1 项必改 + 4 项建议，修改后可进入静态原型生成。

---

## A. 总体判断

这是一份**结构合格但细节有疏漏**的页面卡文档。全部 9 个核心页面都已覆盖，每页都有关键信息和关键操作，状态说明覆盖面不错（多数页面写了 3-5 种状态），UI 规范给出了色彩、布局、组件、导航四类规则，原型三层次说明清晰完整。

主要问题集中在两个方向：**部分页面缺少数据加载失败状态**（会导致原型阶段不考虑异常场景）；**UI 规范缺少字体、图标、加载中状态的约定**（会导致原型中不同页面字体大小不一致）。对静态原型生成来说，这些缺失不会完全阻塞，但会造成原型阶段反复修改。

---

## B. 发现的问题清单

### B1. 缺少字体和排版规范（检查重点 5）

文档第 4 节 UI 规范中只有色彩、布局、组件、导航四类，**没有字体规范**：

- 未定义正文字体大小（是 14px 还是 16px？）
- 未定义标题层级字号（h1/h2/h3 各多大？）
- 未定义字体族（使用 Bootstrap 默认的 system-ui 还是指定具体字体？）
- 未定义行高和字重

**影响**：静态原型阶段，如果没有约定的字号，开发者可能会在 5 个页面里用 3 种不同的 h2 字号，最终到代码阶段才统一调整，浪费迭代次数。

### B2. 缺少数据加载中状态（检查重点 3）

9 个页面没有一个提到"数据加载中"的 UI 状态。至少以下 4 个页面涉及异步数据获取，需要加载中占位：

- **PU-01 用户首页**：统计数据从数据库查询，加载中应显示灰色占位块或 spinner
- **PU-02 座位列表页**：座位数据加载中应显示 loading 提示
- **PA-02 预约管理页**：预约列表加载中
- **PA-04 统计页**：4 个数字卡片加载中

没有这些状态，生成原型时开发者会跳过加载动画，原型看起来"秒开"，到代码阶段才补 loading，交互品质下降。

### B3. 座位管理和新增/编辑表单字段未定义（检查重点 2）

PA-03 座位管理页的"关键信息"只写了"所有座位列表"，"新增/编辑"状态只写了"用 Modal 弹窗承载表单"，但**表单里有哪些字段完全没有写**。

作为参考，PRD Lite 第 11.2 节的 Seats 表定义包含以下字段：
- SeatNumber（座位编号，必填）
- Area（区域，必填，下拉选择）
- Floor（楼层，可选）
- Description（设施描述，可选）
- IsActive（是否启用，默认 true）

缺少这些字段说明，UI 设计者需要自己去查 PRD 的数据表定义，或者等开发阶段才发现表单字段漏了。

### B4. PU-05 状态标签颜色与 UI 规范不一致（检查重点 6）

PU-05 状态说明中写：
> 预约中 = 蓝色标签

但 UI 规范第 4.1 节色彩表中，**没有定义蓝色（#0d6efd）用于状态标签** —— 蓝色被定义为主色（按钮、链接、导航）。状态标签的颜色约定是：
- 成功/空闲 = #198754（绿色）
- 警告/预约中 = #ffc107（橙色）

也就是说：PU-05 说"预约中=蓝色标签"，但 UI 规范说"预约中=橙色"。两个文档互相矛盾。

而且在 PU-02 座位列表中，座位状态"已预约"用的是橙色标签（合理）。但在 PU-05 中，预约记录状态"预约中"如果用蓝色，就和座位列表中的橙色不一致。两种状态本质上一回事（座位已预约 = 该座位有预约中的记录），但用了两个颜色，会让用户困惑。

### B5. PU-01 和 PU-03 缺少数据加载失败异常态（检查重点 3）

| 页面 | 现有状态 | 缺少的状态 |
|------|----------|------------|
| PU-01 用户首页 | 未登录默认态、已登录态 | **数据加载失败**（统计数据查不到时） |
| PU-03 座位详情页 | 空闲态、已预约态 | **座位不存在**（ID 无效或已删除）、**数据加载失败** |

对比之下 PU-02 和 PU-04 都有完善的错误状态覆盖。

### B6. 原型视觉稿优先级的说明可优化（检查重点 6）

第 5 节说"线框图阶段每页 1 张，共 9 张"，但视觉稿阶段"选择 3-5 个核心页面"。为什么从 9 张缩到 3-5 张没有解释。新人看到这里可能会疑惑：是偷懒还是有意为之？建议补充一句原因（如"视觉稿费时，核心页面确定风格后其余页面可类推"）。

## C. 必改项

### C1. 统一预约中/已预约的状态标签颜色，消除矛盾

选择以下方案之一并修改到文档中（**推荐方案一**）：

**方案一（推荐）：全部统一为橙色**

- 座位列表中的"已预约" → 橙色（`#ffc107`）
- 我的预约列表中的"预约中" → 橙色（`#ffc107`）
- 理由：两个状态实质对应同一个业务含义（该座位当前被人占了），一个颜色减少用户认知负担

**方案二：严格区分**

- 座位列表中的"已预约" → 橙色（座位维度的状态）
- 我的预约列表中的"预约中" → 蓝色（`#0d6efd`，预约记录维度的状态）
- 理由：两个维度的状态不同，分开更精确
- 需要同时在 UI 规范 4.1 节增加蓝色标签的用途说明

同时修改两处：PU-05 的状态说明 + UI 规范 4.1 节的色彩表。

## D. 建议优化项

### D1. 补充字体和排版规范

在 UI 规范中新增一节：

```
4.5 字体与排版规范

- 字体族：Bootstrap 默认系统字体栈（system-ui, -apple-system, ...）
- 正文字号：16px（Bootstrap $font-size-base）
- H1：28px / Bold，仅用于页面标题
- H2：20px / Semi-Bold，用于卡片标题
- H3：16px / Semi-Bold，用于区域标题
- 表格字号：14px
- 统计数字：32px / Bold（首页和统计页的卡片数字）
- 行高：1.5（正文）/ 1.2（标题）
```

### D2. 补充 4 个页面的数据加载中状态

在对应页面的状态说明表中新增一行：

| PU-01 用户首页 | 数据加载中 | 统计数据区域显示 3 个灰色占位矩形块，无文字 |
| PU-02 座位列表 | 数据加载中 | 列表区域显示 "正在加载座位数据..." |
| PA-02 预约管理 | 数据加载中 | 表格区域显示 "正在加载预约记录..." |
| PA-04 统计页 | 数据加载中 | 4 个卡片区域显示灰色占位块 |

### D3. 补全新增/编辑座位表单的字段

在 PA-03 座位管理页的"组件"或"状态说明"中补充：

```
新增/编辑弹窗表单字段：
- 座位编号（SeatNumber）：文本框，必填，如 "A-12"
- 区域（Area）：下拉框，必填，选项来自系统预置区域列表
- 楼层（Floor）：文本框，可选，如 "3F"
- 设施描述（Description）：文本框，可选，如 "靠窗、有插座"
- 是否启用（IsActive）：开关/勾选框，默认启用
```

### D4. 补充 PU-01 和 PU-03 的异常状态

| PU-01 | 数据加载失败 | 统计卡片区域显示 "统计数据加载失败，请刷新页面" 的提示条 |
| PU-03 | 座位不存在 | 显示 "座位不存在或已删除"，提供"返回列表"按钮 |
| PU-03 | 数据加载失败 | 显示 "座位信息加载失败，请稍后重试" |

### D5. 视觉稿 3-5 张的选择理由说明

在第 5 节视觉稿说明中补充：

```
注：视觉稿只选择 3-5 个核心页面，而非全部 9 页。
原因是视觉稿阶段主要目的是确定视觉风格和组件样式。
风格确定后，其余页面在线框图基础上套用相同组件即可，无需逐一做视觉稿。
```

## E. 是否可以进入下一步 "静态原型与原型评审"

| 条件 | 状态 |
|------|------|
| 覆盖全部 9 个核心页面 | ✅ 全部覆盖 |
| 每页关键信息与操作清楚 | ⚠️ PA-03 新增/编辑表单字段未定义 |
| 成功/空/异常状态覆盖 | ⚠️ 缺少加载中状态和部分异常态（见 B2、B5） |
| 用户端/管理端布局区分 | ✅ 第 4.4 节和第 6 节清楚区分 |
| UI 规范明确 | ⚠️ 缺少字体规范（见 B1） |
| 原型三层次说明 | ✅ 结构完整（见 B6 可优化） |
| 足够支撑静态原型生成 | ⚠️ 状态标签颜色矛盾（C1）会直接导致原型配色不一致 |

**结论：完成 C1（状态标签颜色统一）后可进入下一步。** C1 是唯一阻塞项——如果原型直接用蓝色标签表示"预约中"，会和 UI 规范中的橙色冲突，到代码阶段才发现不一致就晚了。D 组建议优化项推荐全部采纳，但不强制阻塞。特别是 D1（字体规范）和 D2（加载中状态），趁原型阶段一并补上效率最高。
